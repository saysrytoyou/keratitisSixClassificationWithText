# 角膜炎图文联合细分类实验方案

## 1. 目标定位

本方案作为后续所有实验的统一基准。

本课题的目标不是复用上一篇五分类图像模型，而是在更大、更均衡、且引入病历文本的新数据条件下，提出并验证一个新的图文联合细分类方法。

当前研究问题定义为：

- 如何将病历文本中的临床先验信息有效融入三模态裂隙灯图像判别过程
- 如何提升 6 类角膜炎细分类性能，特别是类别均衡性与临床一致性

后续所有实验、消融、可视化和结论分析，均以本方案为唯一执行基准。

---

## 2. 数据与任务定义

### 2.1 数据范围

当前正式实验仅使用同时具备图像和病历文本的样本。

- 图像目录：`picture`
- 文本目录：`text/角膜炎数据完整版信息.xlsx`
- 实验代码目录：`shiyan`

### 2.2 当前主任务

主任务定义为 6 类图文联合细分类。

6 类标签如下：

1. `A_HSK_EM`：单纯疱疹病毒性角膜炎-上皮/混合型
2. `B_OVK`：其他疱疹病毒性角膜炎
3. `C_BK`：细菌性角膜炎
4. `D_FK`：真菌性角膜炎
5. `E_Normal`：正常
6. `F_NIK`：非感染性角膜炎

### 2.3 当前样本定义

当前清单构建结果如下：

- 总样本数：`1121`
- 图像模态：`DLI / FSI / SBI`
- 每位患者每种模态至少 1 张图像

### 2.4 暂不纳入主任务的样本

- `Z棘阿米巴性角膜炎 8 例`

当前不纳入原因：

- 只有 Excel 病历信息
- 无法与 `picture` 中图像严格对齐
- 不能直接参与当前图文主实验

后续可作为补充分析或扩展任务单独讨论。

## 3. 现阶段基线实验结论

当前已完成的 5 折正式结果如下：

- `image`：`Acc 0.4023`，`Macro-F1 0.3555`
- `text`：`Acc 0.6628`，`Macro-F1 0.6509`
- `late`：`Acc 0.6655`，`Macro-F1 0.6544`
- `prior`：`Acc 0.6646`，`Macro-F1 0.6580`

当前结论只用于确定后续方法方向，不作为最终模型结论。

当前可确认的事实：

1. 病历文本中包含强临床先验信息
2. 简单图文拼接只能带来有限提升
3. 先验引导式融合优于简单晚期融合，尤其在 `Macro-F1` 上更优
4. 当前轻量图像分支不足以代表图像模态真实上限，因此不能据此得出“文本比图像更强”的泛化结论

## 4. 第二篇论文的主方法定位

### 4.1 方法主线

第二篇论文的方法主线固定为：

`Text Prior Encoding -> Modality Reweighting -> Prior-Guided Feature Modulation -> Joint Classification`

### 4.2 不采用的方案

以下方案不作为第二篇论文的主创新：

- 直接复用上一篇 `HCCMF` 作为主体结构
- 仅做 `image + text` 末端拼接
- 再次把 `cross-modal attention` 作为核心创新点

原因：

- 与上一篇方法重合度过高
- 不能充分体现第二篇在图文联合建模上的独立创新

### 4.3 第二篇论文的方法定位

第二篇论文的方法应被定义为：

- 面向角膜炎 6 类细分类
- 面向更大、更均衡数据集
- 面向病历文本先验引导视觉判别
- 面向图文协同建模的新方法

---

## 5. 主模型结构定义

第二篇论文的正式主模型暂定为：

`Clinical Prior Guided Multimodal Fusion Network`

可简称：

- `CPGMF-Net`

### 5.1 模块一：Text Prior Encoder

输入：

- 主诉
- 现病史
- 既往史

输出：

- 文本临床先验向量 `F_text`

作用：

- 将病历信息编码为临床先验，而不是直接用于末端分类

### 5.2 模块二：Modality Reweighting

输入：

- `F_text`
- 三模态图像特征 `F_dli / F_fsi / F_sbi`

输出：

- 三模态动态权重 `w_dli / w_fsi / w_sbi`

作用：

- 根据临床先验，动态调整不同图像模态的重要性

目标：

- 回答不同病例中，哪种图像模态更值得关注

### 5.3 模块三：Prior-Guided Feature Modulation

这是第二篇论文的核心创新模块。

不采用跨模态注意力作为核心方法，改用文本引导的特征调制机制。

优先采用：

- 门控机制
- 或 FiLM 条件调制机制

推荐主方案：

- `FiLM-based Feature Modulation`

基本形式：

- 文本分支生成条件参数 `gamma` 和 `beta`
- 对图像融合特征进行条件调制

作用：

- 让文本临床先验在特征级影响视觉表示学习
- 避免与上一篇基于注意力的图像融合结构重合

### 5.4 模块四：Joint Classification Head

输入：

- 经动态重加权与先验调制后的图文联合特征

输出：

- 6 类分类结果

---

## 6. 主方法创新点固定版

后续所有汇报和实验组织均围绕以下 3 个创新点展开：

### 创新点 1

提出一种面向角膜炎细分类的临床先验引导图文联合诊断框架，将病历文本作为条件先验参与视觉判别。

### 创新点 2

设计文本引导的动态模态重加权机制，使模型能够针对不同病例自适应调节 `DLI / FSI / SBI` 三种图像模态的贡献。

### 创新点 3

设计基于门控或 FiLM 的特征调制模块，使病历文本在特征级而非决策级引导视觉特征学习，提升细粒度分类性能与临床一致性。

## 7. 实验体系固定版

### 7.1 第一层：基础对比实验

这一组实验用于回答“新方法是否有效”。

固定保留以下对比组：

1. `image-only`
2. `text-only`
3. `image + text late fusion`
4. `image + text prior-guided baseline`
5. `proposed CPGMF-Net`

说明：

- 前 4 组是基础对比组
- 第 5 组是正式主方法

### 7.2 第二层：主方法内部消融实验

这一组实验用于回答“新方法的提升来自哪里”。

固定消融项如下：

1. 去掉文本分支
2. 去掉模态重加权
3. 去掉特征调制
4. 仅保留 late fusion
5. 仅保留 modality reweighting
6. 仅保留 feature modulation
7. full model

### 7.3 第三层：文本先验可靠性实验

这一组实验用于回答“文本增益是否可信”。

固定实验内容如下：

1. 原始病历文本实验
2. 清洗后病历文本实验
3. 对比 `text-only / late / prior / proposed`

文本清洗重点检查：

- 直接诊断词
- 病原词
- 培养结果词
- 明显标签提示词
- 明确病种名称

### 7.4 第四层：任务扩展实验

这一层作为补充分析，不早于主方法稳定后开展。

包括：

1. 5 类补充实验
2. 层级分类实验
3. 去除极小样本类后的稳定性分析

---

## 8. 当前固定评价指标

由于类别不完全均衡，后续统一采用以下指标：

1. `Accuracy`
2. `Macro-F1`
3. 每类 `Precision / Recall / F1`
4. 混淆矩阵
5. 各折均值与标准差

结论判断优先级固定为：

`Macro-F1 > 每类 F1 > Accuracy`

原因：

- 当前任务属于细分类任务
- 类别均衡性和难类表现比单纯 Accuracy 更重要

## 12. 当前执行结论

从本方案开始，后续所有实验以以下内容为固定基准：

- 主任务：`6 类图文联合细分类`
- 主方法：`Text Prior Encoding -> Modality Reweighting -> Prior-Guided Feature Modulation -> Joint Classification`
- 主创新：`临床先验引导 + 动态模态重加权 + 特征调制`
- 主评价：`Macro-F1 优先`
- 主实验路线：`基线 -> 主方法 -> 消融 -> 可信性分析 -> 扩展任务`

如后续要修改方法路线、主任务定义或创新点，必须先修改本文件，再开展新的正式实验。
