# CPGMF-Net 技术文档

## 1. 文档目的

本文件只服务于代码实现，不承担论文写作功能。

本文件用于固定以下内容：

- 主模型 `CPGMF-Net` 的模块边界
- 各模块输入输出接口
- 张量维度约定
- 主方案与消融方案的对应关系
- 训练脚本与模型代码的改造原则

后续所有代码实现均以本文件为准。

---

## 2. 主模型总览

主模型全称：

- `Clinical Prior Guided Multimodal Fusion Network`

简称：

- `CPGMF-Net`

主链路固定为：

`Text Prior Encoding -> Modality Reweighting -> Prior-Guided Feature Modulation -> Joint Classification`

输入为：

- 三模态图像：`DLI / FSI / SBI`
- 病历文本：`raw_text`

输出为：

- `6` 类 logits

---

## 3. 输入输出定义

### 3.1 数据输入

单个 batch 的输入格式固定为：

- `image_dli`: `[B, 3, H, W]`
- `image_fsi`: `[B, 3, H, W]`
- `image_sbi`: `[B, 3, H, W]`
- `text`: `[B, T]` 或 `[B, D_text_in]`
- `label`: `[B]`

说明：

- 当前版本中图像来自对同模态多张图取均值后的单张张量
- 当前文本先保持 `TF-IDF` 稠密向量输入
- 后续若切换成 `BERT` 类编码器，训练脚本接口不变，只替换 `Text Prior Encoder`

### 3.2 中间表示维度约定

为减少前期反复改代码，先固定一套默认维度：

- 图像模态特征维度：`D_v = 256`
- 文本先验特征维度：`D_t = 256`
- 联合特征维度：`D_f = 256`

即：

- `F_dli ∈ R^{B x 256}`
- `F_fsi ∈ R^{B x 256}`
- `F_sbi ∈ R^{B x 256}`
- `F_text ∈ R^{B x 256}`
- `F_img ∈ R^{B x 256}`
- `F_fused ∈ R^{B x 256}`

---

## 4. 模块定义

## 4.1 Text Prior Encoder

### 功能

将病历文本编码为临床先验向量。

### 输入

- `text`: `[B, D_text_in]`

### 输出

- `F_text`: `[B, D_t]`

### 当前实现建议

第一版先采用：

- `TF-IDF -> MLP`

即：

- 训练脚本仍用 `TfidfVectorizer`
- 文本编码器内部用两层 `Linear + ReLU + Dropout`

### 后续可替换实现

后续可扩展为：

- `Chinese Medical BERT`
- `MacBERT`
- 结构化病历字段编码器

但这些不是第一阶段必须项。

---

## 4.2 Image Encoders

### 功能

分别编码三种裂隙灯图像模态。

### 输入

- `image_dli`: `[B, 3, H, W]`
- `image_fsi`: `[B, 3, H, W]`
- `image_sbi`: `[B, 3, H, W]`

### 输出

- `F_dli`: `[B, D_v]`
- `F_fsi`: `[B, D_v]`
- `F_sbi`: `[B, D_v]`

### 设计原则

- 三个模态先独立编码
- 编码器结构参数可以共享或不共享
- 第一版为了简单稳定，先采用“共享 backbone + 不同输入”

### 当前实现建议

第一版可采用：

- 轻量 CNN backbone

后续再升级为：

- `ResNet18 / ResNet50`
- `ConvNeXt-Tiny`
- `EfficientNet`

第一阶段不要求立刻用最强 backbone，先把方法链路跑通。

---

## 4.3 Modality Reweighting

### 功能

基于文本先验，动态生成三模态权重。

### 输入

- `F_text`: `[B, D_t]`
- `F_dli`: `[B, D_v]`
- `F_fsi`: `[B, D_v]`
- `F_sbi`: `[B, D_v]`

### 输出

- `w_dli`: `[B, 1]`
- `w_fsi`: `[B, 1]`
- `w_sbi`: `[B, 1]`
- `F_img`: `[B, D_v]`

### 计算形式

建议第一版采用：

`w = softmax(MLP(F_text))`

其中：

- `w = [w_dli, w_fsi, w_sbi]`

然后：

`F_img = w_dli * F_dli + w_fsi * F_fsi + w_sbi * F_sbi`

### 说明

这一模块负责回答：

- 这个病例下，哪一种图像模态更重要

该模块是第二篇与第一篇图区分的重要部分之一。

---

## 4.4 Prior-Guided Feature Modulation

### 功能

使用文本先验在特征级调制图像融合表示。

### 输入

- `F_text`: `[B, D_t]`
- `F_img`: `[B, D_v]`

### 输出

- `F_fused`: `[B, D_f]`

### 主方案

主方案固定为：

- `FiLM-based Feature Modulation`

### 计算形式

先由文本先验生成：

- `gamma = MLP_gamma(F_text)`，shape `[B, D_v]`
- `beta = MLP_beta(F_text)`，shape `[B, D_v]`

再调制图像融合特征：

`F_fused = gamma ⊙ F_img + beta`

其中：

- `⊙` 表示逐元素乘法

### 设计理由

- 与上一篇的 attention 路线区分明显
- 比单纯门控更灵活
- 便于做消融

### 备选消融方案

备选方案为：

- `Gate-based Modulation`

形式：

`g = sigmoid(MLP(F_text))`

`F_fused = g ⊙ F_img`

说明：

- `Gate` 作为消融
- `FiLM` 作为主方法

---

## 4.5 Joint Classification Head

### 功能

对联合表示做最终分类。

### 输入

- `F_fused`: `[B, D_f]`

### 输出

- `logits`: `[B, 6]`

### 当前实现建议

采用：

- `Linear -> ReLU -> Dropout -> Linear`

作为统一分类头。

---

## 5. 主模型前向流程

主模型前向流程固定为：

1. 文本输入经过 `Text Prior Encoder` 得到 `F_text`
2. 三模态图像分别经过 `Image Encoders` 得到 `F_dli / F_fsi / F_sbi`
3. `Modality Reweighting` 根据 `F_text` 生成动态模态权重，并得到 `F_img`
4. `Prior-Guided Feature Modulation` 根据 `F_text` 和 `F_img` 得到 `F_fused`
5. `Joint Classification Head` 输出最终 logits

---

## 6. 消融映射表

后续代码必须支持以下消融切换：

### 6.1 基线模型

1. `image`
2. `text`
3. `late`
4. `prior`

### 6.2 主方法家族

5. `cpgmf_gate`
6. `cpgmf_film`

其中：

- `cpgmf_film` 是正式主模型
- `cpgmf_gate` 是重要对照消融

### 6.3 模块消融

后续需要支持以下模块开关：

1. `use_modality_reweighting = False`
2. `use_feature_modulation = False`
3. `modulation_type = gate / film / none`

推荐最终映射如下：

- `late`：无重加权，无调制，直接拼接
- `prior`：无模态重加权，仅简单 gate 式引导
- `cpgmf_gate`：有重加权，有 gate 调制
- `cpgmf_film`：有重加权，有 FiLM 调制

## 7. 代码改造原则

## 7.1 不再把所有模型写在一个训练脚本里

后续模型代码应拆分到 `models/` 目录。

建议结构如下：

- `shiyan/models/__init__.py`
- `shiyan/models/encoders.py`
- `shiyan/models/fusion.py`
- `shiyan/models/cpgmf.py`

### 文件职责

`encoders.py`

- `ImageEncoder`
- `TextPriorEncoder`

`fusion.py`

- `ModalityReweighting`
- `GateModulation`
- `FiLMModulation`

`cpgmf.py`

- `ImageOnlyModel`
- `TextOnlyModel`
- `LateFusionModel`
- `PriorGuidedModel`
- `CPGMFNet`

---

## 7.2 训练脚本的职责

`run_experiments_torch.py` 后续只负责：

- 读 manifest
- 划分 fold
- 构造 dataloader
- 创建模型
- 训练与验证
- 结果保存
- 可视化输出

模型细节不再写在训练脚本内部。

---

## 7.3 当前阶段不做的改造

当前阶段暂不做：

- BERT 文本编码器接入
- patch/token 级视觉 Transformer
- 复杂 region-level attention
- report generation

当前目标只有一句话：

**先把 CPGMF-Net 的主方法链路稳定实现，并完成正式实验对比。**

---

## 8. 当前优先实现顺序

第一优先级：

- 搭建 `models/` 结构
- 把现有 `image / text / late / prior` 从训练脚本中拆出来

第二优先级：

- 实现 `ModalityReweighting`
- 实现 `GateModulation`
- 实现 `FiLMModulation`
- 实现 `CPGMFNet`

第三优先级：

- 在训练脚本中增加 `cpgmf_gate / cpgmf_film`
- 跑通 debug 实验

第四优先级：

- 跑正式实验
- 生成可视化结果
